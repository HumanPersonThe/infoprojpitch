Job Overview

This patch addresses a session expiry defect in the HALO ITSM Connector where OAuth2 access tokens were not being refreshed before expiry, causing connector sessions to silently fail after 60 minutes of inactivity. Affected users experienced lost change request submissions and delayed incident routing.

This is a temporary patch pending a full fix from the HALO vendor in their next API release.

Business Impact: Eliminated reported session drop incidents. Connector uptime improved from 94% to 99.1% following deployment.

Logic & Process
1
Token Expiry Detection
The connector now checks token expiry 5 minutes before the scheduled expiry time rather than at the point of the next API call. A background refresh task is triggered when remaining lifetime falls below this threshold.
2
Retry Queue
If a token refresh fails due to a transient network error, the request is placed in a local retry queue with exponential backoff (2s, 4s, 8s). After 3 failed retries the connector logs a critical error and alerts the Service Desk via HALO incident.
3
Temporary Workaround Note
The root cause lies in the HALO API v4 token endpoint's undocumented 60-minute hard expiry. The permanent fix requires the HALO vendor to extend token TTL. This patch will be superseded by v2.1.0 upon vendor resolution.
